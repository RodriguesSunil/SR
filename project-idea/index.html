Project Idea - Sunil Rodrigues

Overview

I have chosen a project that will help my organisation in management and deployment of SAP Plant Maintenance Training. Over the years we have had various versions of LMS (Learning Management Systems) and these systems though good require a lot of customisations to make them do what you need to do. This gets quite expensive and requires constant updating as items and training materials change.

With my implementation of SAP at the organisation I work, I had set up all the training materials in a SQL database with a Dotnet front end application. This was done for me by one of my System Engineers. I want to harness this application and build a web front end to capture the users in the organisation, what roles they hold to determine what training would be required for them to complete. The front end would record what training the users have completed and create a career progression pathway for them.

Motivation

Any system implemented is as good as the users using it and the data that you capture. Over the years I have worked at various mining companies and the common theme that come out is that SAP is quite time consuming to learn and hence people shy away from it. One of the reasons for this is that we try to teach the users everything, which is not ideal for the user base that is not used to the learning techniques that we use. Hence, I want to build an application that gives the user the ability to choose the skills they need, to do his job first and then enhance his skills to improve himself so they become more marketable.

There are many mining companies using SAP and most are in the same situation. They currently rely on SAP Trainers to come to site to run courses. Having an application that is self-learning, will reduce the cost of labour and also give the users a better training platform.

Description

The application will be based on mobile technology and available on any browser, after authenticating using Azure. Users in the organisation are already within Azure, so I would need to link them to their credentials. A Mind map showing the flow of data has been made. This is the initial map and will be updated as more development is progressed.

The below items will be included as part of the overall application.

  • User Access – this will be governed by Azure AD with MFA and their corresponding SAP Roles and Positions
  • Training Materials – this will be developed for each of the transactions that the user will undertake. It will simply follow the steps training using the web application and SAP training client.
  • Learning Outcomes – each position will have core competencies set up using a stepped approach. Users will progress from the lower levels to higher levels with each core competency achieved recorded against their logins.
  • SQL Database – will store the competencies achieved and the master data required to run the application. This will be exposed to Power BI for reporting.
  • Dashboard – this will be a Power BI window that will show the competencies achieved for the department, sub-department and crew. It will aid the management to see how good their teams are with respect to achieving their competencies.
  • SAP – the position and user roles assigned to the users will be exposed to the application. This will assist the application to show the training and competencies required for individual user. The SAP Organisation structure will be used to match users to department and line managers.
  • Chat – chat will be available using AI to pick out key answers to queries that the user may have. A dictionary of key answers will be maintained and updated as the users' queries get recorded. The ability to talk to a Subject Matter Expert (SME) will be available.
  • SharePoint – will store the physical training catalogue and the videos used for training. This will ensure the latest documentation is available for the users.
The training will be based on competency levels and the users will be able to progress themselves to the next level once they have completed each level. The users will be able to check in SAP what they have done as the client in which they practice will be kept live.

Tools and Technologies

There would be a fair amount of connectivity between various existing applications. Our organisation stores all documents in SharePoint Online and Teams so this would need to be connected to the application. User roles and position from SAP and Azure Active Directory will need to be migrated. The web application would need to be exposed to the users using multifactor authentication (MFA) so that data within our current applications are not compromised. Current training materials will need to be revamped to make them mobile friendly. New ways to find the correct training when needed using QR codes will be implemented.

Skills Required

Looking through the requirements for the application, the following skills will be required to successfully complete the project.

  • Web Developer
  • Software Engineer
  • SAP Functional Consultant
  • BI Analyst
  • Database Administrator
  • Program/Project Manager
  • Cost Controller
  • Learning and Development Consultant

Outcome

The project will take some time to bed in and get working. The initial learning from testing and piloting a small user base will be necessary so improvements can be made. A key part will be in the reporting, as this will help me to make changes where it matters the most. Successful implementation will improve the data capture and errors in the system. The application also then could be scalable to other types of training such as operations or health and safety.
